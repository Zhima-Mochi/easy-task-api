# easy-task-api - Domain Driven Design Example

## Introduction
This is a simple example of a Domain Driven Design (DDD) project. The project is a restful task API application.

## Run
```bash
make docker-build
make docker-run
```

## Endpoints
- GET /tasks
- POST /tasks
- GET /tasks/{id}
- PUT /tasks/{id}
- DELETE /tasks/{id}

### Swagger
- GET /swagger/index.html
